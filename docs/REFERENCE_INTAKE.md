# Reference Intake

References are design pressure, not source templates.

## Borrow

- Vocabulary.
- Failure modes.
- Interface boundaries.
- Fixture shapes.
- Validation discipline.

## Reject

- Copying code without license and provenance review.
- Adding dependencies just because a reference uses them.
- Turning reference runtime behavior into core requirements.
- Importing private names, local paths, package ids, or tuning constants.

Any future reference note should record the lesson borrowed, the overreach
rejected, the Manifold layer affected, and the validation follow-up.
